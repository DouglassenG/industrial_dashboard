## Tarefa — Padronizar botão de som com o estilo do botão de tema

**Arquivo alvo:** `@apps/web/app/components/layout/Header.tsx`

### Ponto de partida

Substitua o componente `<Button />` do shadcn/ui abaixo:

```tsx
<Button
  variant="outline"
  size="sm"
  onClick={handleToggleSound}
  aria-label={soundEnabled ? "Desativar som" : "Ativar som"}
  className="whitespace-nowrap flex-shrink-0"
>
  {soundEnabled ? (
    <BellRing size={14} aria-hidden="true" />
  ) : (
    <Bell size={14} aria-hidden="true" />
  )}
  <span>{soundEnabled ? "Som Ativo" : "Ativar Som"}</span>
</Button>
```

### Pelo modelo abaixo (botão de tema já existente no Header):

```tsx
<button
  onClick={onToggleDark}
  aria-label={darkMode ? "Ativar modo claro" : "Ativar modo escuro"}
  className="inline-flex items-center gap-1.5 px-3 py-1.5 rounded-md bg-gray-100 dark:bg-gray-700 text-gray-800 dark:text-white text-sm transition-all duration-300 min-w-[80px] whitespace-nowrap flex-shrink-0 hover:-translate-y-0.5 hover:shadow-md active:scale-95"
>
  {darkMode ? (
    <>
      <Sun size={14} aria-hidden="true" />
      <span>Light</span>
    </>
  ) : (
    <>
      <Moon size={14} aria-hidden="true" />
      <span>Dark</span>
    </>
  )}
</button>
```

### Resultado esperado

O botão de som deve ser um `<button>` nativo com **exatamente as mesmas classes
Tailwind** do botão de tema. Apenas os seguintes valores devem ser adaptados:

| Propriedade   | Botão de tema        | Botão de som                   |
| ------------- | -------------------- | ------------------------------ |
| `onClick`     | `onToggleDark`       | `handleToggleSound`            |
| `aria-label`  | modo claro/escuro    | "Ativar som" / "Desativar som" |
| Ícone ativo   | `<Sun size={14} />`  | `<BellRing size={14} />`       |
| Ícone inativo | `<Moon size={14} />` | `<Bell size={14} />`           |
| Texto ativo   | `"Light"`            | `"Som Ativo"`                  |
| Texto inativo | `"Dark"`             | `"Ativar Som"`                 |

### Regras

- COPIE as classes Tailwind do botão de tema sem alteração.
- REMOVA o import do `<Button />` do shadcn/ui se não for mais usado no arquivo.
- NÃO altere nenhum outro elemento do Header.
- NÃO altere a lógica `handleToggleSound` existente.
- MANTENHA o `aria-label` dinâmico conforme o estado `soundEnabled`.
