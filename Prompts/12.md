## Tarefa — Refinar sombras e cores do projeto

**Arquivos alvo:**

- `@apps/web/app/globals.css`
- `@apps/web/app/components/` (todos os componentes)

---

### Objetivo

Tornar o dashboard visualmente mais vivo, refinando **exclusivamente** sombras
e cores existentes. Nenhum layout, posicionamento ou lógica deve ser alterado.
Aplique tecnicas sutis de web design em paineis de automacao.

---

### Etapa 1 — Leitura obrigatória antes de qualquer alteração

1. Leia `@apps/web/app/globals.css` por completo e mapeie:
   - Todas as variáveis CSS de cor (`--color-*`, `--background`, `--foreground`, etc.)
   - Todas as classes de sombra já aplicadas (`shadow-sm`, `shadow-md`, etc.)

2. Percorra `@apps/web/app/components/` e registre:
   - Quais cards, badges e containers usam `shadow-*`
   - Quais elementos usam `border`, `ring`, ou `bg-*` para separação visual

Só inicie as alterações após concluir esse mapeamento.

---

### Etapa 2 — Sombras

Aplique os upgrades abaixo, **preservando o tamanho e intensidade sutis**:

| Situação atual     | Substituir por                                   |
| ------------------ | ------------------------------------------------ |
| `shadow-sm`        | `shadow-md`                                      |
| `shadow-md`        | `shadow-lg`                                      |
| Sem sombra em card | `shadow-sm`                                      |
| Hover sem sombra   | `hover:shadow-md transition-shadow duration-300` |
| Hover com sombra   | `hover:shadow-lg transition-shadow duration-300` |

Regras:

- NUNCA aplique `shadow-xl` ou `shadow-2xl` — o efeito deve ser sutil.
- Em dark mode, adicione `dark:shadow-black/30` junto à sombra existente para
  profundidade sem exagero.

---

### Etapa 3 — Cores

Torne as cores ligeiramente mais saturadas e contrastantes, sem sair da paleta
já definida em `globals.css`:

| Elemento             | Ajuste sugerido                                          |
| -------------------- | -------------------------------------------------------- |
| Bordas de cards      | Subir um tom (ex: `border-gray-200` → `border-gray-300`) |
| Texto secundário     | Subir um tom (ex: `text-gray-400` → `text-gray-500`)     |
| Badges de status     | Adicionar `ring-1 ring-inset` com a cor correspondente   |
| Fundo de cards hover | `hover:bg-gray-50 dark:hover:bg-gray-800/60`             |
| Indicadores ativos   | Garantir contraste mínimo de 3:1 com o fundo             |

Regras:

- NÃO troque famílias de cor (ex: não mude `blue` para `indigo`).
- NÃO altere cores de texto principal — apenas secundário e de suporte.
- USE somente classes Tailwind existentes — sem valores arbitrários
  (ex: `bg-[#1a1a2e]`).
- CONSULTE `globals.css` para garantir que nenhuma variável CSS seja
  sobrescrita de forma inconsistente.

---

### O que NÃO fazer

- NÃO altere `font-size`, `font-weight`, `padding`, `margin` ou `gap`.
- NÃO mova ou reposicione nenhum elemento.
- NÃO altere lógica, hooks ou dados.
- NÃO instale dependências.
- NÃO crie novos componentes.
- NÃO aplique `drop-shadow` via CSS inline.
