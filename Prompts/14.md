## Tarefa — Reorganizar Header no mobile

**Arquivo alvo:** `@apps/web/app/components/layout/Header.tsx`

---

### Regras invioláveis

- USE exclusivamente classes Tailwind — sem CSS inline.
- NÃO altere a estrutura ou lógica dos elementos existentes.
- NÃO adicione, remova ou renomeie componentes.
- NÃO altere o layout em tablet (`md:`) e desktop (`lg:`) — apenas mobile
  (`< 640px`) está com problema.
- CONSULTE `globals.css` antes de aplicar qualquer cor.

---

### Problema identificado

No mobile, o Header exibe todos os elementos em linha sem hierarquia, causando:

- Título cortado (`Dashboard de...`)
- Badge "Conectado" e botões sobrepostos sem espaçamento
- Ausência de alinhamento com os cards abaixo

---

### Solução esperada

Reorganize o Header mobile em **duas linhas bem definidas**:

**Linha 1 — Identidade**

```
[ ícone ] [ Título do Dashboard ]    [ badge Conectado ]
```

- `flex items-center justify-between w-full`
- Título: `truncate` para evitar corte abrupto
- Badge: alinhado à direita com `ml-auto`

**Linha 2 — Ações**

```
[ Ativar Som ]   [ Dark ]
```

- `flex items-center gap-2 w-full`
- Botões com `flex-1` para ocupar a largura igualmente no mobile
- Alinhe com a margem lateral dos cards abaixo usando o mesmo `px-*`
  já aplicado no container principal da página

---

### Classes de breakpoint a usar

```tsx
// Container do Header
className="flex flex-col gap-2 px-4 py-3
           sm:flex-row sm:items-center sm:justify-between sm:gap-4"

// Linha 1 — Título + Badge
className="flex items-center justify-between w-full gap-2"

// Título
className="font-semibold text-sm truncate min-w-0"

// Linha 2 — Botões de ação
className="flex items-center gap-2 w-full sm:w-auto"

// Cada botão na linha de ações (mobile)
className="flex-1 sm:flex-none justify-center"
```

---

### Alinhamento com elementos adjacentes

- Identifique o `px-*` e `mx-*` usados nos cards imediatamente abaixo
  do Header em `@apps/web/app/page.tsx`.
- Aplique o mesmo valor de padding horizontal no container do Header para
  garantir alinhamento visual entre header e conteúdo.

---

### O que NÃO fazer

- NÃO altere `sm:`, `md:` ou `lg:` existentes — apenas adicione ou corrija
  o comportamento base (mobile-first, sem prefixo).
- NÃO troque os componentes de botão — apenas adicione classes de largura.
- NÃO altere cores ou sombras.
- NÃO mova o Header para fora do seu container atual.
