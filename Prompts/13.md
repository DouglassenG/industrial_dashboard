## Tarefa — Aplicar paleta de cores STW no dashboard

**Arquivos alvo:**

- `@apps/web/app/globals.css`
- `@apps/web/app/components/layout/Header.tsx`
- `@apps/web/app/components/` (demais componentes)

---

### Regras invioláveis

- USE exclusivamente classes Tailwind — sem CSS inline, sem valores arbitrários.
- CONSULTE `globals.css` antes de qualquer alteração para não sobrescrever
  variáveis já em uso.
- NÃO altere layout, posicionamento, tipografia, lógica ou dados.
- NÃO crie nem remova componentes.
- NÃO instale dependências.

---

### Paleta de referência (extraída do site stw.com.br)

| Token            | Hex       | Classe Tailwind | Uso                             |
| ---------------- | --------- | --------------- | ------------------------------- |
| Azul marinho     | `#002F6C` | `blue-900`      | Header, elementos primários     |
| Azul primário    | `#0078D4` | `blue-600`      | Destaques, badges, links ativos |
| Fundo claro      | `#F5F5F5` | `gray-100`      | Background geral (light mode)   |
| Texto principal  | `#1A1A1A` | `gray-900`      | Títulos e textos de destaque    |
| Texto secundário | `#555555` | `gray-600`      | Labels, subtítulos              |
| Fundo de cards   | `#FFFFFF` | `white`         | Cards e painéis                 |
| Borda sutil      | `#E0E0E0` | `gray-200`      | Divisórias e bordas de cards    |

Declare essas cores como variáveis CSS em `globals.css` antes de consumir nos
componentes:

```css
:root {
  --color-header-bg: theme("colors.blue.900");
  --color-primary: theme("colors.blue.600");
  --color-bg: theme("colors.gray.100");
  --color-text: theme("colors.gray.900");
  --color-text-muted: theme("colors.gray.600");
  --color-card: theme("colors.white");
  --color-border: theme("colors.gray.200");
}
```

---

### Etapa 1 — Header

Em `@apps/web/app/components/layout/Header.tsx`:

- Aplique `bg-blue-900` como cor de fundo do Header.
- Textos e ícones dentro do Header devem usar `text-white`.
- Botões dentro do Header devem usar `hover:bg-blue-800 transition-colors
duration-200` para feedback sutil de interação.
- Em dark mode, mantenha `dark:bg-blue-950` para diferenciar levemente.

---

### Etapa 2 — Cards e painéis

Em todos os componentes de `@apps/web/app/components/`:

- Fundo dos cards: `bg-white dark:bg-gray-800`
- Borda dos cards: `border border-gray-200 dark:border-gray-700`
- Sombra padrão: `shadow-sm hover:shadow-md transition-shadow duration-300`
- Título dos cards: `text-gray-900 dark:text-white`
- Texto secundário: `text-gray-600 dark:text-gray-400`

---

### Etapa 3 — Destaques e badges

- Badges de status ativo: `bg-blue-600 text-white`
- Badges de alerta: mantenha a cor semântica existente (vermelho/amarelo),
  apenas garanta que o fundo seja `blue-600` apenas para status "online/ativo".
- Links e elementos interativos: `text-blue-600 hover:text-blue-700`

---

### O que NÃO fazer

- NÃO use `blue-500` ou inferior para o header — o azul marinho é `blue-900`.
- NÃO troque cores semânticas (verde = ok, vermelho = erro, amarelo = alerta).
- NÃO aplique `blue-900` fora do Header — use `blue-600` para destaques
  internos nos cards.
- NÃO altere dark mode além do especificado acima.
